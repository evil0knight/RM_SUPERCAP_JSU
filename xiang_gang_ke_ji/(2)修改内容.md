> 第二批买回来的原件,INA电流检测不准,具体点的问题是每次上电,I_A_CONVERT_BIAS都会变,基准值会变,光靠拟合是没用的,所以需要写一个补丁

> 所以我在每次上电的时候,读取IA和IB为0的时候的SampleManager::adcIA.sum和SampleManager::adcIB.sum

```
宏展开：
I = GAIN * (sum / 8) + BIAS
当 I = 0 时：
0 = GAIN * (sum_zero / 8) + BIAS
=> BIAS = -GAIN * (sum_zero / 8)
  所以只要在 I 确实为 0 的时候，读出 sum_zero，就能算出当次上电的 BIAS。
```

```
为什么 GAIN 不漂？
  GAIN 对应的是传感器的灵敏度（每安培对应多少 ADC 计数），这由硬件电阻、参考电压决定，温漂很小。
  为什么 BIAS 会漂？
  BIAS 对应的是零点电压，INA 运放的输入失调电压每次上电会有微小变化，导致 I=0 时 ADC 读数不固定，所以每次上电重新测一次零点。
```

```
前提条件：
校准时被测电流必须真的是 0
iA、iB 满足（buckboost 未工作）
iReferee 不满足（因为超电被供电的瞬间,电流是从电源端reference流向底盘chassis的,电流无法为0,除非先把连接底盘的xt30拔出来），所以不能动态校准
```
